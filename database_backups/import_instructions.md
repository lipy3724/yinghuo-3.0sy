# 数据库导入说明

## 问题分析
原始备份文件导入失败的原因：
1. 包含了本地特有的 `migration_history` 表
2. 服务器数据库可能使用不同的迁移系统
3. 密钥长度限制问题

## 解决方案

### 方案1：使用修复后的备份文件（推荐）
使用文件：`yinghuo_backup_fixed_20251023.sql`
- 已移除 migration_history 表
- 保留所有业务数据和表结构

### 方案2：分步导入
1. 先导入结构：`yinghuo_structure_backup_20251023_114405.sql`
2. 再导入数据：`yinghuo_data_backup_20251023_114405.sql`

### 方案3：手动处理
如果仍有问题，可以：
1. 先在服务器上创建空的 migration_history 表
2. 然后导入完整备份

## 导入命令示例
```bash
# 在服务器上执行
mysql -u用户名 -p数据库名 < yinghuo_backup_fixed_20251023.sql
```

## 注意事项
- 导入前请确保目标数据库为空或做好备份
- 如果遇到字符集问题，请确保数据库使用 utf8mb4 字符集
- 如果遇到密钥长度问题，可能需要调整 MySQL 配置
